# 项目经验

## 某某科技股份有限公司 (2022.01 - 至今)

### 高并发电商交易系统重构 (核心项目)

**角色**：核心全栈架构师

**技术栈**：Java 17, Spring Cloud Alibaba, Go, Redis, Kafka, Kubernetes, MySQL

**项目描述**：针对公司原有单体交易系统进行微服务拆分，解决大促期间系统响应缓慢、数据库连接过载的业务瓶颈，支撑起大促期间核心交易链路平稳运行。

**核心职责与产出**：
- **微服务重构设计**：将原有复杂单体拆分为交易、支付、库存、优惠券等多个高内聚微服务，使用 gRPC 实现微服务间的同步调用，解耦核心链路。
- **高并发与缓存调优**：引入 Redis 缓存与热点本地缓存，结合 Canal 监听 MySQL binlog 实现缓存在线同步更新。大幅度削减数据库查询压力，核心接口响应时间由 **220ms 降至 18ms**。
- **流控防刷与熔断降级**：引入 Sentinel 进行细粒度接口限流、熔断及防刷治理，实现核心微服务集群可用率达 **99.99%**。

### 全栈研发效能度量看板

**角色**：全栈开发

**技术栈**：Node.js (NestJS), React, Tailwind CSS, ClickHouse, Docker

**项目描述**：为提升产研团队的交付效率，设计并实施全流程效能度量系统，对接 Gitlab、Jira、Jenkins 等多源数据，提供可视化的效能分析大盘。

**核心职责与产出**：
- **多源数据管道**：基于 Webhook 与定时任务设计统一的接入层，将团队的代码提交、需求流转、构建部署数据实时同步至 ClickHouse。
- **响应式度量大屏**：使用 React + Tailwind CSS 开发极具科技感的现代化看板，包含多维度折线图、极坐标图及交付漏斗。

## 某互联网独角兽企业 (2019.06 - 2021.12)

### 全链路灰度发布系统

**角色**：高级全栈开发

**技术栈**：Go, Spring Boot, Istio, Kubernetes, Vue.js, Prometheus

**项目描述**：为保障业务线平稳升级，开发统一的灰度控制平台，实现基于特定用户画像、IP网段及流量比例的精准流量分发与路由。

**核心职责与产出**：
- **Istio 控制面集成**：基于 Kubernetes CRD 以及 Go-client 动态管理 Istio 的 VirtualService 和 DestinationRule，保证流量调配达到毫秒级下发。
- **灰度发布大盘**：基于 Vue.js 设计开发配置后台，提供一键发布、渐进式放量、实时指标监控以及一键秒级回滚功能。

### 分布式智能调度系统

**角色**：核心开发

**技术栈**：Java, Quarkus, Netty, ZooKeeper, PostgreSQL

**项目描述**：支撑全网大促活动期间海量定时任务的可靠分发与调度，单日处理任务达 **千万级**。

**核心职责与产出**：
- **无状态调度节点**：设计基于 ZooKeeper 的动态选主与任务分片机制，实现调度节点无状态化与横向弹性扩容。
- **高性能通信层**：基于 Netty 开发自定义通信协议，保障调度任务的分发时延在 **10ms** 以内。
